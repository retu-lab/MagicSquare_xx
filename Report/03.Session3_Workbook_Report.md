## Mom Test 입력 (요약)

| 항목 | 내용 |
|------|------|
| **페르소나** | 4×4 **부분 마방진**(빈칸 2개)을 손 계산·코드 과제(ECB 분류)로 다루는 **학습자** |
| **진짜 문제 (한 문장)** | 4×4 부분 마방진을 채운 뒤 행·열 합이 맞아 보이면, 대각선 검증을 끝까지 하지 않은 채 ‘계산 실수’만 의심하며 20분 가까이 같은 틀린 답 주변을 헤맨다. |
| **Mom Test 증거 3줄** | ① 빈칸 2개를 넣은 뒤 행·열 합은 34로 맞아 보였지만, 대각선 하나를 빼먹어 약 20분을 허맸다. ② 그 20분 동안 대각선은 직접 더해 보지 않았고, 문제 원인을 계산 실수로만 의심하며 찾았다. ③ 대각선을 처음 더해 본 시점은 20분이 지난 뒤였고, 그때서야 계산이 이상하다는 걸 알았다. |

## 1) 주제 한 문장 (Mom Test 기반, 솔루션 최소화)

**행·열 합이 34로 맞아 보여도 대각선(2개)을 끝까지 확인하지 않고 ‘계산 실수’만 의심하며, 틀린 답 주변에서 시간을 쓰는 검증 습관을 줄인다.**

> 솔루션(자동 검증기·UI)이 아니라 **사람의 검증 순서·판단 착각**이 주제이다.

---

## 2) R-G-I-O

| | 내용 |
|---|------|
| **Role** | 4×4 부분 마방진(ECB) 과제를 푸는 **학습자** + 검증 단계에서 “끝까지 확인했는지”를 짚어 주는 **AI 페어** |
| **Goal** | 빈칸 2개를 채운 4×4 격자에 대해 **행 4 + 열 4 + 대각선 2** 합(=34)을 **빠짐없이** 확인하고, 하나라도 미검증·불합격이면 “완료/맞음”이라고 말하지 않는다 |
| **Input** | 4×4 격자(일부 채워짐 + 빈칸 2개 후보 값), 목표 합 **34**, (선택) 행·열만 34인 **함정** 격자 |
| **Output** | 10개 줄(행·열·대각선) 각각의 **합·합격/불합격·미검증** 목록; “행·열 OK”만으로는 성공 판정 불가 |

---

## 3) 성공 기준 3개 (Mom Test 증거와 연결)

| # | 성공 기준 | 연결 증거 |
|---|-----------|-----------|
| **1** | 행·열이 모두 34여도 **두 대각선 검증 전**에는 “완료” 또는 “맞음” 판정을 내리지 않는다 | 증거 ① — 대각선 하나 빠뜨려 20분 헤맴 |
| **2** | 행·열 통과 + 대각선 미검증(또는 불합격) 입력에 대해, “다시 더해 봐”가 아니라 **대각선 확인·불일치**를 먼저 알린다 | 증거 ② — 20분 동안 대각선은 안 보고 계산 실수만 의심 |
| **3** | 검증 순서 **행 → 열 → 대각선1 → 대각선2**가 고정되고, Test Loop의 **의도적 대각선 누락/오류 케이스**에서 반드시 잡힌다 | 증거 ③ — 대각선 확인은 20분 뒤에야 처음 |

## 5) 8계층 — 이번 세션에서 만드는 것만

| 계층 | 이번 세션 산출물 | Mom Test 연결 |
|------|------------------|---------------|
| **Rule** | `.cursor/rules/` (또는 동등 규칙): “행·열만 34면 끝” 금지, **10줄 합** 필수, 미검증 줄 명시 | 증거 ①·③ — 대각선 빠뜨림 방지 |
| **Command** | `/verify-grid` 등: 4×4 격자 입력 → **행·열·대각선 2** 합 표 + 미검증/불합격만 강조 | 증거 ② — 계산 실수 의심 전 **대각선부터** 짚음 |
| **(Skill)** | *(선택)* `SKILL.md`: 4×4 부분 마방진 **검증 순서·함정 케이스** 재사용 | 반복 과제에서 동일 20분 함정 재현 방지 |
| **Test Loop** | 함정 격자 1개 이상(행·열=34, **대각선 ≠34** 또는 **대각선 미검증 시나리오**) + Rule/Command 출력이 성공 기준 1–3을 만족하는지 확인 | 증거 ①–③ 전부를 **재현 가능한 테스트**로 고정 |

**이번 세션에서 만들지 않음:** Subagent, Hook, MCP, Canvas, Automation 등 나머지 8계층.

## 멘토 코멘트

Mom Test의 **20분 함정**(행·열 OK → 대각선 미확인 → 계산 실수만 의심)을, **Rule + Command + (Skill) + Test Loop**로 “끝까지 검증” 행동에 묶는 세션이 목표이다. 솔루션(Validator·UI)을 먼저 만들지 말고, **성공 기준 3개가 Test Loop에서 재현되는지**로 검증한다.
